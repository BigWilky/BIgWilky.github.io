127.Word ladder

Description:

Link is Here

Given two words ( beginWord and endWord ), and a dictionary’s word list, find the length of shortest transformation sequence from beginWord to endWord , such that:

  1. Only one letter can be changed at a time.
  2. Each transformed word must exist in the word list.

Note:

  • Return 0 if there is no such transformation sequence.
  • All words have the same length.
  • All words contain only lowercase alphabetic characters.
  • You may assume no duplicates in the word list.
  • You may assume beginWord and endWord are non-empty and are not the same.

Example 1:

Input:
   beginWord = "hit",
    endWord = "cog",
    wordList = ["hot", "dot", "dog", "lot", "log", "cog"]

Output : 5

Explanation : As one shortest transformation is "hit" -> "hot" -> "dot" -> "dog" -> "cog", return its length 5.

Solution:

BFS + Vector

Code is here.

int ladderLength(string beginWord, string endWord, vector<string>& wordList) {
    queue<string> list;
    int ladder = 1;
    list.push(beginWord);
    while (!list.empty())
    {
        for (int n = list.size();  n>0; n--)
        {
            string word = list.front();
            list.pop();
            if (word == endWord)
            {
                return ladder;
            }
            for (int j = 0; j < word.size(); j++)
            {
                char save = word[j];
                word[j] = '*';
                for (int k = 0; k < wordList.size(); k++)
                {
                    string temp = wordList[k];
                    char save_temp = temp[j];
                    temp[j] = '*';
                    if (temp == word)
                    {
                        temp[j] = save_temp;
                        list.push(temp);
                        wordList.erase(wordList.begin() + k);
                        k--;
                    }
                }
                word[j] = save;
            }


        }
        ladder++;
    }
    return 0;
}

        if (res[i] == 0) {//Recursive stop condition
            res[i] = 1;
            for (auto j : al[i]) {
                int cnt1[26] = {};
                dfs(j, l, al, cnt1, res);
                for (auto k = 0; k < 26; ++k)
                    cnt[k] += cnt1[k];
            }
            res[i] = ++cnt[l[i] - 'a'];
        }
    }
    vector<int> countSubTrees(int n, vector<vector<int>>& edges, string labels) {

        vector<vector<int>> al(n);

            al[e[1]].push_back(e[0]);

###PS:
#### Array vs Vector


Arrays:

- builtin language construct;
- provide just a contiguous, indexable sequence of elements
- are of fixed size; you can't resize an array in C++
- their size must be a compile-time constant unless they are allocated dynamically
- they take their storage space depending from the scope where you declare them;


std::vector:

- is a template class;
- is implemented as a dynamic array;
- grows and shrinks dynamically;
- can be copied/assigned (this performs a deep copy of all the stored elements);

For execution efficiency

Array>vector. The main reason is that the expansion process of vector consumes a lot of time.




set and Unordered set

- set is based on red black tree, so it's ordered at any time.
- unordered_set is based on hash table.so the time complexity of data insertion and search is very low.It is almost constant time, but the price is more memory consumption
